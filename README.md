# sprint-9-project
Репозиторий для разработки проекта 9-го спринта.
Ссылка на регистри в облаке: cr.yandex/crp12ttpj5jom8ph6qsc

Бизнес-идея: Architech планирует запустить тегирование пользователей в приложении на основе статистики по заказам.

Функциональные требования:
1)входные данные в формате JSON
2) DWH на Data Voult STG DDS DMC
3) В CDM нужны две витрины. Первая витрина — счётчик заказов по блюдам; вторая — счётчик заказов по категориям товаров.

Нефункциональные требования:
1) Информация по заказам — это потоковые данные, которые будут передаваться через брокер сообщений. 
Нагрузка на систему заказов — 5 заказов в минуту. на старте загрузят историю за неделю, т. е. почти 50 000 заказов сразу. Надо обеспечить идемпотентность обработки сообщений из брокера. 
То есть при повторной обработке сообщения состояние БД не должно измениться.
2)Суммарный бюджет на весь проект не должен превышать 5000 рублей в месяц.
Нужно оставить возможность переходить из одного облака в другое.
3) В DWH данные будут передаваться через Kafka.
Обрабатываем поток данных из Kafka и записываем в PostgreSQL.
В сообщениях есть id, которые необходимо будет перевести в имена. Например, блюда, рестораны, пользователи. Для этого тебе понадобится Valkey.
4) Обработка сервисов на Python и нужно развернуть их в Kubernetes.
По одному сервису на слой. Каждый сервис будет читать поток данных из Kafka и формировать свой слой данных: STG-Service, DDS-Service и CDM-Service.

Для визуализаций дашбордов - DataLens: https://datalens.ru/3m0qnq58pls2n
графики:

датасет 1 на основании витрины cdm.user_category_counters
1) круговая диаграмма, показывающая популярность категорий блюд на основе доли пользователей, которые заказывали блюда этой категории на основании витрины 
2) круговая диаграмма, показывающая популярность категорий блюд на основе доли заказов, в которых были блюда этой категории;

датасет 2 на основании витрины cdm.user_product_counters
3) столбчатая диаграмма, показывающая популярность блюд по количеству уникальных пользователей, заказавших блюдо;
4) столбчатая диаграмма, показывающая популярность блюд по количеству заказов, в которых было блюдо.
