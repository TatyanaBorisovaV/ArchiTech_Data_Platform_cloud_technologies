### ArchiTech Data Platform
### Описание проекта

Проект по построению аналитической платформы для сегментации пользователей на основе истории заказов в приложении ArchiTech.

Решение реализует потоковую обработку данных заказов, построение хранилища данных по методологии Data Vault и формирование аналитических витрин для дальнейшей визуализации в DataLens.

### Цель проекта
Подготовить инфраструктуру для тегирования пользователей на основе их предпочтений и истории заказов.

### Архитектура решения

Источник событий
↓
Apache Kafka
↓
STG Service
↓
DDS Service
↓
CDM Service
↓
PostgreSQL (DWH)

Для хранения данных используется многослойная архитектура Data Vault:

STG — слой загрузки исходных данных;
DDS — детальный слой хранения данных;
CDM — слой аналитических витрин.

Каждый слой реализован отдельным сервисом на Python и развернут в Kubernetes.

### Особенности реализации
Реализована потоковая обработка данных заказов через Kafka (входные данные в формате JSON).
Настроена загрузка данных в DWH на базе PostgreSQL.
Реализована идемпотентная обработка сообщений для исключения дублирования данных при повторной обработке событий.
Для обогащения данных используются справочники в Valkey, позволяющие преобразовывать идентификаторы пользователей, ресторанов и блюд в бизнес-атрибуты.
Сервисы контейнеризированы (1 сервис - 1 контейнер) и развернуты в Kubernetes.
Архитектура предусматривает возможность миграции между облачными провайдерами.

### Аналитические витрины
cdm.user_product_counters
Витрина содержит агрегированную статистику по блюдам:
- количество заказов по каждому блюду;
- количество уникальных пользователей, заказавших блюдо.

cdm.user_category_counters
Витрина содержит агрегированную статистику по категориям блюд:
- количество заказов по категориям;
- количество пользователей, заказывавших блюда определённой категории.

### Визуализация данных
Данные визуализируются в DataLens.
Дашборд DataLens https://datalens.ru/3m0qnq58pls2n

Построены следующие графики:

На основе витрины cdm.user_category_counters круговые диаграммы:
  распределение пользователей по категориям блюд;
  распределение заказов по категориям блюд.
На основе витрины cdm.user_product_counters столбчатые диаграммы:
  популярность блюд по количеству уникальных пользователей;
  популярность блюд по количеству заказов.

### Результат
* Построена облачная аналитическая платформа для обработки потоковых данных заказов, включающая Kafka, Kubernetes, PostgreSQL и Data Vault-хранилище. 
* Реализованы аналитические витрины и дашборд для анализа предпочтений пользователей и последующего построения системы тегирования.
